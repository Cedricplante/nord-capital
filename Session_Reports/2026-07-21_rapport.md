### 3. Bugs corrigés (les 6 de l'audit)
| # | Sévérité | Problème | Fix |
|---|----------|----------|-----|
| 1 | Critical | `positionValueCAD` (snapshot.js) : `totalSize` invalide/NaN → propagé jusqu'à l'upsert Supabase → `value: null` silencieux dans `portfolio_history` | Garde-fous `Number.isFinite()` partout + erreur explicite (500 loggé) si `totalCAD` non-fini au lieu d'écrire une valeur corrompue |
| 2 | Warning | `userData.currency` lu mais jamais utilisé pour convertir le cash → faux si le compte n'est pas en CAD | `cashToCAD()` dans le nouveau module partagé, fetch du taux `{DEVISE}CAD=X` si nécessaire |
| 3 | Warning | `snapshot.js` et `monthly-report.js` recalculaient la valeur du portfolio avec des formules différentes (shares dérivées de `totalSize/avgEntry` vs `p.shares` direct) → risque de divergence entre le graphique et l'email mensuel | Créé `api/_lib/valuation.js` — source unique partagée par les deux fichiers |
| 4 | Info | `api/check-alerts.js` public, sans auth ni rate-limit → n'importe qui pouvait spammer le Resend | Vérification du token Supabase (doit correspondre à `SUPABASE_USER_ID`) → 401 sinon |
| 5 | Warning | Dédup "1 email d'alertes/jour" reposait sur `localStorage` (par appareil) → doublons possibles en alternant portable/PC | Dédup déplacée côté serveur (`user_data.last_alert_date`, nouvelle colonne Supabase) |
| 6 | Info | Petit code mort (fallbacks snake_case jamais utilisés dans `positionValueCAD`) | Noté, pas critique, laissé tel quel |

## Décisions importantes
- **Module partagé plutôt que duplication** : au lieu de corriger le cash/la valorisation séparément dans les deux fichiers cron, création de `api/_lib/valuation.js` (convention Vercel : dossier `_`-préfixé exclu du routing) pour qu'ils ne puissent plus diverger à l'avenir.
- **Dédup alertes côté serveur plutôt que client** : choix de résoudre le bug localStorage et le bug de sécurité (endpoint public) en une seule fois via la même colonne Supabase — plus robuste que deux fixes séparés.
- **Ne pas exécuter git depuis le sandbox sur le mount** : après plusieurs `.lock` résiduels non-supprimables, décision de toujours passer par PowerShell (Cédric) pour les opérations git, Claude se limitant à l'édition de fichiers.
